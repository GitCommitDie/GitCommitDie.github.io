FAQ

Why is it taking so long/making so many extra requests?

TL;DR: It needs to or things will break.

To ensure there are no gaps in the timeline, there needs to be a buffer between the previous timestamp and the next timestamp. That's also why all before/after requests have a fixed size of 100. Without this multiple posts made within the same second could be missed.

Beyond that, it needs to fetch the same number of items per each type, so in some cases combined searches will have to overfetch by quite large margins in order to avoid missing data.

All of these problems would be solved if there was a combined submission/comment endpoint and working before_id/after_id parameters but alas there is not.
